# Output Layout

Given `--output /path/to/out_dir`, the pipeline writes a deterministic directory layout.

## Where To Look First

- Ranked results: `results/`
- Step outputs and intermediates: `output/`
- Logs: `logs/`

## Root Files and Directories

- `config/`: per-step YAML configs generated by `configure`
- `inputs/`: derived inputs written during `configure` (e.g. concatenated target PDB, chain maps, rewritten frameworks)
- `steps.yaml`: manifest of pipeline steps
- `pipeline_input.json`: normalized input (resume identity)
- `pipeline_state.json`: run metadata + tool stamps (identity enforcement for resume)
- `.work/<step>/queue.db`: work-queue progress ledger
- `.ppiflow_lock/`: run lock directory (prevents concurrent controllers)
- `.orchestrator/`: orchestrator plan/status (when using multi-GPU / worker pools)

## Output Mode

Default output mode is `minimal`. Use `--output-mode full` to preserve heavy intermediates.

Minimal mode keeps only artifacts needed for resume and ranking. AF3 CIFs are always kept under:
- `output/af3score_round*/cif/`
- `output/af3_refold/cif/`

In `--skip-refold` mode, ranking derives final structures from `output/af3score_round2/cif/` and materializes PDBs under `results*/structures/cache/` for downstream consumers that expect PDBs.

## Optional Files

When output mode is `full`, or when step configs enable optional retention, additional intermediates can be preserved under `output/_optional/`.

See `src/pipeline/prune.py` for the definitive optional keys.

## Heartbeat / Status Files

- `status.json` and `status_rank*.json` are written to the output root during execution.
- Set `PPIFLOW_HEARTBEAT=0` to disable.
- Interval: `PPIFLOW_HEARTBEAT_INTERVAL` (default 30s).
